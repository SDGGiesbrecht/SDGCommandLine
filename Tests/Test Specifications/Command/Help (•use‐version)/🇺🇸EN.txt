$ tool help •no‐colour

tool serves as an example tool.

Subcommands

demonstrate‐text‐formatting demonstrates text formatting.
empty‐cache removes any cached data.
execute demonstrates successful execution.
fail demonstrates a failure.
help displays usage information.
reject‐argument [unsatisfiable] demonstrates rejection of an argument.
set‐language [language preference] sets the language preference. (Omit the argument to revert to the system preferences.)
version displays the version in use.

Options

•language [language preference] A language to use instead of the one specified in preferences.
•no‐color Removes color from the output.
•use‐version [version] Attempts to download and temporarily use the specified version insead of one which is installed.

Argument Types

[language preference] A list of IETF language tags or SDGCornerstone language icons. Semicolons indicate fallback order. Commas indicate that multiple languages should be used. Examples: “en-US” or “🇺🇸EN” → American English, “nv,en;es” → both Navajo and English, otherwise Spanish
[version] A version number or “development”.
